# Task

Consider yourself a Senior Go developer.
I am writing a configuration app. A single configuration is called a Tegn, and it can be installed.
I would like to create the ability to uninstall it. However, uninstalling is a destructive operation.
For example, if I create a local-config file, I can remove it to uninstall it, but that file might have been modified by the user; therefore, removing it would delete user data, which is not ideal.

I would like you to analyze my Tegns and identify which Tegn uninstallations may lead to user data loss (or other destructive outcomes). 
Consider how to resolve these issues.

Your output MUST use the following format: an h2 markdown heading for the file name (the Tegn's file path).
An h3 header "Destructive" with a list of destructive things that may happen, or the word "NOTHING" if uninstallation is safe. Include an h3 header with considerations on how to make destructive actions more user-friendly, or skip this h3 if there are no destructive outcomes.
