# Task

Consider yourself a Senior Go developer.
I am writing a configuration app. A single configuration is called a Tegn, and it can be installed.
I would like to create the ability to uninstall it. 

The user must be able to specify "install" or "uninstall."
I have an action type (Install, Uninstall) and `ParseActionTypeFromArgv1` to handle it.
Currently, the installation logic is located in the `mainInternal` function.
I would like you to split the installation and uninstallation logic into separate functions.

I have a UI to install Tegns and Tegnsetts. I would also like a UI to uninstall Tegns and Tegnsetts. It must:
- Display only the installed Tegns in the correct order (as a flat, one-level list without grouping them into Tegnsetts).
- Allow the user to select Tegns to uninstall.
- Exclude the parameters editor.
- Include a Help function to show information about the Tegn.

Your output MUST use the following format: an h2 markdown heading for the file name (the file path to be created), and the markdown code block containing the file content.
