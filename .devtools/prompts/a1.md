# Prompt

Consider yourself a Go developer.
Write an e2e test for me to check the correctness of osinfo harvesting. 
The "test" consists of:
A docker-compose configuration for various distributions:
- Alpine Linux
- Debian
- Rocky Linux

Inside, it uses a Python script to verify the output. It uses the following technique:
- Retrieve the value from the Go output and attempt to "prove" it (for example, if sysLib is glibc, find the glibc .so file, and so on).

Your output MUST use the following format: an h2 markdown heading for the file name (the file path to be created), and the markdown code block containing the file content.


# Output example 

```json
{"arch":{"value":"amd64","raw":"amd64"},"osType":{"value":"linux","raw":"linux"},"pkgManager":{"value":"pacman","raw":"pacman"},"sysLib":{"value":"glibc","raw":"glibc"},"distrib":{"id":"arch","id_like":"","name":"Arch Linux","version":{"value":{"major":0,"minor":0,"patch":0},"raw":""},"unknown":false}}
```
